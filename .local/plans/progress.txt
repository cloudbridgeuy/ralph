## OutputBlock System

### Enum Structure
```rust
#[non_exhaustive]
enum OutputBlock {
    Text(TextBlock),
    ToolInvocation(ToolInvocationBlock),
    ToolResult(ToolResultBlock),
    Separator,
}
```

### ToolInvocationVariant (6 variants)
- `Bash { command, description }` - Shell commands
- `Grep { pattern, path, output_mode, glob, case_insensitive, type_ }` - Search (uses GrepInvocationBuilder)
- `Read { file_path, offset, limit }` - File reading
- `Glob { pattern, path }` - File pattern matching
- `TodoWrite { todos: Vec<TodoItem> }` - Task management
- `Default { content }` - Fallback for unknown tools

### ToolResultVariant (12 variants)
- `Bash { content, exit_code }` - Shell output
- `EditBeforeAfter { file_path, before, after }` - Edit with before/after content
- `EditDiff { file_path, diff }` - Edit with unified diff
- `EditNoChanges { file_path, message }` - No-op edit
- `WriteNewFile { file_path, content }` - New file creation
- `WriteOverwrite { file_path, before, after }` - File overwrite
- `WriteNoChanges { file_path }` - No-op write
- `Read { file_path, content, line_count, truncated }` - File content
- `Grep { content, match_count, output_mode }` - Search results
- `Glob { content, file_count, truncated }` - Matched files
- `TodoWrite { message }` - Task update confirmation
- `NotebookEdit { notebook_path, cell_id, operation, content, old_content }` - Jupyter edits
- `Default { content }` - Fallback

### Design Decisions
- Variants capture **data** not rendered strings (re-rendering with different settings)
- Uses `#[serde(tag = "type", rename_all = "snake_case")]` for clean TOML
- `#[non_exhaustive]` on all enums for forward compatibility
- `GrepInvocationBuilder` for ergonomic 6-field construction

## Data Flow

### Live Execution
1. StreamProcessor accumulates `Vec<OutputBlock>` during execution
2. Each event handler pushes blocks as they render to stdout
3. `finish()` returns `StreamProcessorResult` with output_blocks
4. `IterationLog` receives output_blocks, serializes to `iteration-N.toml`

### Serialization (TOML)
```toml
[[output_blocks]]
type = "tool_result"
tool_name = "Read"
is_error = false

[output_blocks.result_type]
type = "read"
file_path = "src/main.rs"
content = "fn main() { ... }"
line_count = 50
truncated = false
```

### Replay
1. `replay.rs` reads `iteration-N.toml`, deserializes `Vec<OutputBlock>`
2. Falls back to `chunks` for older sessions without output_blocks
3. `ReplayRenderer` renders each block using shared `crate::render` functions
4. Output visually identical to live execution

## Interrupt Handling

### SIGTERM/SIGINT Persistence
1. Signal sets `INTERRUPTED` flag
2. Subprocess drains output, calls `processor.finish()`
3. Returns `SubprocessError::Interrupted { partial_result }`
4. `RunError::Interrupted` carries `partial_result` and `pending_before`
5. Main loop writes partial `iteration-N.toml`
6. Session finalized as "interrupted"

**Result**: Partial sessions can be replayed showing output up to interruption.

## Spinner Line Clear Fix (2026-01-20)

### Summary
Fixed spinner display to properly clear the line before each redraw, preventing residual characters when the text length changes.

**Location**: `crates/ralph/src/spinner.rs`

### Problem
The spinner used `\r` (carriage return) to return to the start of the line but didn't clear the remaining content. When the spinner text got shorter (e.g., when time changed from "59s" to "1m 0s", or when the context message changed from "Summarizing progress file..." to "Thinking..."), residual characters from the previous longer line would remain visible.

### Solution
Added `\x1b[K` (ANSI escape code for "clear to end of line") immediately after `\r` in the spinner output format.

**Before:**
```rust
let spinner_line = format!(
    "\r\x1b[36m{}\x1b[0m {} {}",
    spinner_char, message, time_display
);
```

**After:**
```rust
let spinner_line = format!(
    "\r\x1b[K\x1b[36m{}\x1b[0m {} {}",
    spinner_char, message, time_display
);
```

### ANSI Escape Codes Reference
| Code | Meaning |
|------|---------|
| `\r` | Carriage return (move cursor to start of line) |
| `\x1b[K` | Erase in line (clear from cursor to end of line) |
| `\x1b[36m` | Set text color to cyan |
| `\x1b[0m` | Reset all text formatting |

### Acceptance Criteria Met
- ✅ Spinner format uses '\r\x1b[K' to clear the line before each redraw
- ✅ No leftover characters appear when time format changes (e.g., '59s' → '1m 0s')
- ✅ No leftover characters appear when spinner context changes
- ✅ Spinner continues to display correctly in terminal output

## Ask Command - Session Persistence (2026-01-22)

### Summary
Added session creation and persistence for the `ask` command. Each invocation now creates a session that can be viewed with `ralph sessions` and replayed with `ralph replay`.

**Location**: `crates/ralph/src/ask.rs` and `crates/ralph/src/main.rs`

### Implementation Details

**Updated `AskConfig` struct:**
```rust
pub struct AskConfig {
    pub prompt: String,
    pub timeout_secs: u64,
    pub theme_config: ThemeConfig,
    pub verbose_tools: VerboseToolsConfig,
    pub project_path: PathBuf,  // NEW: Absolute path to the project directory
    pub slug: Option<String>,   // NEW: Optional user-provided session slug
}
```

**New `AskResult` struct:**
```rust
pub struct AskResult {
    pub slug: String,  // The session slug that was created
}
```

**New error variant:**
```rust
pub enum AskError {
    // ... existing variants ...
    Session(SessionError),  // NEW: Session initialization errors
}
```

**Session creation flow:**
1. Validate prompt is not empty
2. Call `session::initialize_session()` to create session directory and metadata
3. Store prompt text in `session.toml` for replay
4. Build subprocess config with session info for spinner display
5. Invoke subprocess and return result with session slug

   - Added `Session(SessionError)` variant to `AskError`

   - Updated `ask()` to call `session::initialize_session()` before subprocess
   - Spinner now shows session info (slug, iteration 1/1)
   - Added 5 new tests for session-related functionality

2. **`crates/ralph/src/main.rs`**:
   - Updated `execute_ask()` to pass `project_path` (current working directory)
   - Display session creation message with replay command hint

## Ask Command - Iteration Logging (2026-01-22)

### Summary
Added iteration logging for replay compatibility in the `ask` command. Each ask session now writes an `iteration-1.toml` file that captures all output blocks, tool calls, chunks, and metadata, enabling faithful replay with `ralph replay`.

**Location**: `crates/ralph/src/ask.rs`

### Implementation Details

**New error variant:**
```rust
pub enum AskError {
    // ... existing variants ...
    Iteration(#[from] IterationError),  // NEW: Iteration log write failures
}
```

**New imports:**
```rust
use crate::iteration::{
    write_iteration_log, Chunk, IterationError, IterationLog, LogMetadata, LogToolCall,
};
use chrono::Utc;
```

**New pure function `build_iteration_log`:**
```rust
fn build_iteration_log(
    result: &StreamingSubprocessResult,
    started_at: DateTime<Utc>,
    completed_at: DateTime<Utc>,
) -> IterationLog
```

Extracts from `StreamProcessorResult`:
- `LogMetadata` from `metadata` and `costs` (session_id, model, cost_usd, duration_ms, usage)
- `LogToolCall` from `tool_interactions` (id, name, input, result, is_error)
- `Chunk` from `chunks` (prose, code, diff with language hints)
- `OutputBlock` from `output_blocks` (for faithful replay)

**Updated `ask` function flow:**
1. Capture `started_at` timestamp before session initialization
2. Initialize session (creates `session.toml`)
3. Invoke subprocess with spinner
4. Capture `completed_at` timestamp
5. Build iteration log using `build_iteration_log()`
6. Write iteration log to session directory using `write_iteration_log()`
7. Check exit code (after writing log to preserve partial data on failure)

**Iteration log structure for ask:**
- `sequence: 1` (ask always creates iteration 1)
- `pending_before: 0` (ask has no stories)
- `pending_after: 0` (ask has no stories)
- All other fields populated from subprocess result

### Changes Made
1. **`crates/ralph/src/ask.rs`**:
   - Added imports for iteration module types

   - Added `build_iteration_log()` pure function
