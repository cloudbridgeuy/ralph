## Session Finalization for Ask Command (Story 5)

Implemented session finalization and summary display for the `ask` command:

### Changes Made

1. **AskResult expanded** (`crates/ralph/src/ask.rs`):
   - Added `exit_code`, `cost_usd`, `duration_ms`, `input_tokens`, `output_tokens` fields
   - Removed `NonZeroExit` error variant (exit code now returned in result)
   - Added `extract_metrics_from_log()` helper to extract metrics from iteration log

2. **AskSummary struct** (`crates/ralph/src/startup/types.rs`):
   - New struct for ask command summary display: slug, success, cost_usd, duration_ms, input_tokens, output_tokens

3. **Display functions** (`crates/ralph/src/startup/`):
   - `display_ask_summary()` in display.rs with terminal detection
   - `display_ask_summary_terminal()` in terminal.rs with colors and status icons
   - `display_ask_summary_plain()` in plain.rs for piped output

4. **execute_ask() updated** (`crates/ralph/src/main.rs`):
   - Calls `session::finalize_session()` with Completed/Failed based on exit code
   - Displays `AskSummary` with metrics after execution
   - Returns error after finalization for non-zero exit codes

### Key Design Decisions

- Session finalization happens even on failure (so `ralph sessions` shows correct status)
- Metrics display happens even on failure (user sees cost/duration)
- Finalization errors are logged as warnings, not fatal
- Exit code check happens after log write and finalization to preserve partial data

### Usage

```bash

# Error handling
```

### Key Design Decisions

- Continuation mode uses existing session directory, no new session created
- Sequence number determined by counting existing iteration-*.toml files
- `iteration_count` returned in result for proper session finalization
- `--session` flag serves dual purpose: name new session OR specify session to continue
- Most recent session determined by `started_at` timestamp, filtered by project

## Theme Inheritance for Ask Command (Story 8)

Added `--theme` and `--no-background` flags to the `ask` subcommand for controlling syntax highlighting.

### Changes Made

1. **AskArgs updated** (`crates/ralph/src/cli/mod.rs:172-188`):
   - Added `theme: Option<String>` field with `#[arg(long, value_name = "NAME")]`
   - Added `no_background: bool` field with `#[arg(long)]`
   - Doc comments describe theme resolution precedence and config file support

2. **execute_ask() updated** (`crates/ralph/src/main.rs:504-510`):
   - Theme config now uses `from_config_and_env().merge_cli()` pattern
   - Follows same resolution precedence as `run` and `replay` commands
   - Priority: CLI flag > environment variable > config file > default

3. **Tests added** (`crates/ralph/src/cli/tests.rs`):
   - `test_ask_with_theme` - long flag with theme name
   - `test_ask_with_no_background` - boolean flag
   - `test_ask_without_theme_flags` - default behavior
   - `test_ask_with_theme_and_no_background` - combined flags
   - `test_ask_theme_with_file_path` - custom .tmTheme file path

### Usage

```bash
# Use default theme (from config file, env var, or built-in default)
ralph ask 'hello'

# Override with specific built-in theme
ralph ask --theme 'Monokai Extended' 'hello'
ralph ask --theme 'Solarized (dark)' 'hello'

# Use custom theme file
ralph ask --theme /path/to/custom.tmTheme 'hello'

# Disable background colors
ralph ask --no-background 'hello'

# Combine theme and no-background
ralph ask --theme 'Solarized (dark)' --no-background 'hello'
```

### Theme Resolution Precedence

1. CLI flags (`--theme`, `--no-background`) - highest priority
2. Environment variables (`RALPH_THEME`, `RALPH_NO_BACKGROUND`)
3. Config file (`~/.config/ralph/config.toml` under `[theme]` section)
4. Built-in default (`base16-ocean.dark`) - lowest priority

## Remaining CLI Flags for Ask Command (Story 9)

Added `--timeout`, `--verbose-tools`, and `--no-prompt` flags to the `ask` subcommand.

### Changes Made

1. **AskArgs updated** (`crates/ralph/src/cli/mod.rs:190-221`):
   - Added `timeout: u64` field with `#[arg(long, default_value_t = 600)]`
   - Added `verbose_tools: Option<String>` with `#[arg(long, value_name = "TOOLS", num_args = 0..=1, default_missing_value = "*")]`
   - Added `no_prompt: bool` field with `#[arg(long)]`
   - Doc comments describe timeout behavior, verbose tools syntax, and prompt suppression

2. **execute_ask() updated** (`crates/ralph/src/main.rs:488-550`):
   - Parses `VerboseToolsConfig` from CLI arg with warning output for unknown tools
   - Displays prompt using `startup::display_prompt()` when `!args.no_prompt && !prompt.is_empty()`
   - Passes `timeout_secs` and `verbose_tools` to `AskConfig` (no longer uses `..Default::default()`)

3. **Tests added** (`crates/ralph/src/cli/tests/ask.rs`):
   - `test_ask_with_timeout` - timeout flag with custom value
   - `test_ask_timeout_default` - default 600 seconds
   - `test_ask_with_verbose_tools_all` - enable verbose for all tools
   - `test_ask_with_verbose_tools_specific` - specific tools with `=` syntax
   - `test_ask_without_verbose_tools` - default behavior
   - `test_ask_with_no_prompt_flag` - boolean flag
   - `test_ask_without_no_prompt_flag` - default behavior
   - `test_ask_with_all_new_flags` - combined new flags
   - `test_ask_with_all_flags_combined` - all ask flags together

4. **CLI tests refactored**:
   - Split `cli/tests.rs` into `cli/tests.rs` + `cli/tests/ask.rs` to stay under 1000 lines
   - Ask tests now in separate file referenced via `#[path = "tests/ask.rs"]`

### Usage

```bash
# Custom timeout (default 600 seconds)
ralph ask --timeout 60 'hello'

# Enable verbose output for all tools
ralph ask 'hello' --verbose-tools

# Enable verbose for specific tools
ralph ask --verbose-tools=grep,bash 'hello'

# Suppress prompt display
ralph ask --no-prompt 'hello'

# Combine all flags
ralph ask --timeout 120 --verbose-tools=read --no-prompt 'hello'
```

### Key Design Decisions

- Timeout flag uses same default as `run` command (600 seconds)
- Verbose tools uses same syntax as `run` command (`*` for all, comma-separated for specific)
- Prompt display happens before subprocess invocation (consistent with `run` command)
- Prompt is only displayed when non-empty and `--no-prompt` is not set
