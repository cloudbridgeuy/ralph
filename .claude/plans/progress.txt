### Implementation details
- Bridge between Functional Core (`ralph_core::chunk::ParsedChunk`) and Imperative Shell (`ralph::iteration::Chunk`)
- `Chunk::from_parsed_chunk` uses pattern matching on `ChunkType` enum to create appropriate Chunk variant
- Language field only serialized when present (via `skip_serializing_if`)
- TOML output format matches architecture spec:
  ```toml
  [[chunks]]
  type = "prose"
  content = "I'll implement the feature..."

  [[chunks]]
  type = "code"
  language = "rust"
  content = "fn authenticate() {\n    // ...\n}"

  [[chunks]]
  type = "diff"
  content = "--- a/file.rs\n+++ b/file.rs\n..."
  ```
